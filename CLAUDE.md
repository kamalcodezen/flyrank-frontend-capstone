## Coding Conventions

- Follow responsive design principles.
- Prefer reusable components.
- Follow Conventional Commits.
- Explain changes before making major refactors.

---

# Project Rules Learned from FE-03

## Rule 1

Always keep HTML, CSS, and JavaScript in separate files unless the project explicitly requires otherwise.

## Rule 2

Every form must include proper accessibility support, including labels, ARIA attributes, keyboard navigation, and visible focus states.

## Rule 3

Before considering AI-generated code complete, manually verify validation logic, responsive behavior, browser functionality, and console errors.

## Rule 4

Use detailed prompts with clear constraints, expected behavior, and a verification step instead of vague prompts.
